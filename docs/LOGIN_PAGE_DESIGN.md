## 1. 목적

이 문서는 Figma에 정의된 단일 로그인 화면을 Flowify 프론트엔드에 반영하기 위한 설계 문서다.

이번 작업의 목표는 아래와 같다.

- 현재 `LoginPage`를 Figma 시안과 최대한 비슷한 구조로 재정비한다.
- 기존 로그인 기능은 유지하되, 화면 밀도와 시각 계층을 단순하고 명확하게 정리한다.
- 프로젝트 컨벤션에 맞게 Chakra UI props 기반 구현 방향을 정의한다.
- 백엔드 주도 OAuth 플로우와 충돌 없이 사용할 수 있는 UI 기준을 남긴다.

## 2. 현재 상태 분석

### 2-1. 현재 구현 상태

현재 `src/pages/login/LoginPage.tsx`는 이미 Google 로그인 기능을 수행하고 있다.

현재 페이지의 특징은 아래와 같다.

| 항목 | 현재 상태 |
|---|---|
| 기능 | Google 로그인 버튼 클릭 시 `buildGoogleLoginStartUrl()`로 이동 |
| 레이아웃 | 워드마크 + 카드의 중앙 정렬 구조 |
| 배경 | `bg.surface` |
| 카드 폭 | `maxW="437px"` |
| 텍스트 | 설명 문구 제거, 제목만 유지 |
| 버튼 | Google 아이콘이 포함된 border형 단일 버튼 |
| 상태 처리 | pending 시 spinner + 문구 변경 |
| 에러 처리 | 예외 시 카드 하단 에러 메시지 표시 |

### 2-2. 현재 구현과 남은 차이

현재 로그인 페이지는 Figma 구조를 대부분 반영했다. 남아 있는 차이는 아래와 같다.

| 항목 | 현재 구현 | 보완 여지 |
|---|---|---|
| Google 아이콘 자산 | `react-icons/fc` 사용 | Figma 자산과 1:1 일치 여부 추가 검토 가능 |
| 로딩 상태 | spinner + 문구 표시 | 시안에는 로딩 상태가 정의돼 있지 않음 |
| 콜백 페이지 톤 | `/login`만 우선 정리 완료 | `/auth/callback`과의 시각 일관성은 후속 작업 |

즉, 현재 화면은 로그인 진입 UI 기준으로는 시안과 거의 같은 수준이며, 남은 작업은 세부 polish 성격에 가깝다.

## 3. Figma 시안 분석

### 3-1. 화면 구조

Figma 노드 `1805:1150`은 단순한 중앙 정렬 로그인 화면이다.

전체 구조는 아래와 같다.

```text
전체 화면
└── 세로 중앙 정렬 컨테이너
    ├── Flowify 워드마크
    └── 로그인 카드
        ├── 제목: 로그인
        └── Google 로그인 버튼
```

### 3-2. 주요 시각 수치

Figma 기준으로 확인된 핵심 수치는 아래와 같다.

| 항목 | 값 |
|---|---|
| 워드마크 폰트 크기 | `48px` |
| 워드마크와 카드 간격 | `48px` |
| 카드 너비 | `437px` |
| 카드 내부 패딩 | `48px` |
| 카드 radius | `20px` |
| 카드 border | `#f2f2f2` |
| 카드 shadow | `0 4px 4px rgba(0,0,0,0.25)` |
| 카드 제목 크기 | `24px` |
| 카드 내부 요소 간격 | `10px` |
| 버튼 수평 패딩 | `64px` |
| 버튼 수직 패딩 | `10px` |
| 버튼 border | `#efefef` |
| 아이콘 크기 | `24px` |
| 버튼 텍스트 크기 | `16px` |

### 3-3. 시안 해석

이 화면은 설명 문구나 추가 액션 없이, 사용자를 빠르게 Google 인증으로 진입시키는 용도의 최소 화면이다.

따라서 시안의 핵심은 아래 3가지다.

1. 브랜드명과 로그인 카드만 남긴 단순한 수직 구조
2. 시선이 바로 카드 중앙으로 모이는 넓은 여백
3. 강한 CTA 대신 가벼운 보더형 로그인 버튼

## 4. 구현 범위

### 4-1. 포함 범위

- `/login` 페이지 레이아웃 재구성
- 중앙 정렬 워드마크 배치
- 카드형 로그인 UI 반영
- Google 로그인 버튼 스타일 변경
- 기존 `buildGoogleLoginStartUrl()` 기반 로그인 기능 유지
- 버튼 클릭 중 pending 상태 반영
- 로그인 시작 실패 시 에러 메시지 유지

### 4-2. 제외 범위

- 소셜 로그인 추가
- 회원가입/비밀번호 찾기 등 추가 액션
- 배경 일러스트 또는 별도 브랜딩 그래픽
- OAuth 플로우 로직 변경
- `/auth/callback` 페이지 수정

## 5. UI 설계

### 5-0. 시안 반영 기준

이 화면은 단순한 인증 진입 화면이므로, 프로젝트 공통 토큰을 우선 사용하되 Figma 정확도가 중요한 값은 예외적으로 고정값을 사용한다.

| 항목 | 적용 기준 |
|---|---|
| 배경/텍스트 | semantic token 우선 사용 (`bg.surface`, `text.primary`, `text.secondary`) |
| border/shadow | Figma 시안과 차이가 커질 수 있으므로 raw 값 사용 허용 |
| 스타일링 방식 | Chakra UI props만 사용 |

### 5-1. 전체 레이아웃

페이지는 전체 뷰포트 기준 수직/수평 중앙 정렬로 구성한다.

| 항목 | 설계 기준 |
|---|---|
| 최소 높이 | `100dvh` |
| 정렬 방식 | `alignItems="center"`, `justifyContent="center"` |
| 배경 | `bg.surface` |
| 외곽 여백 | 모바일 기준 좌우 `24px` 이상 |

### 5-2. 상단 워드마크

워드마크는 카드 바깥에 독립적으로 배치한다.

| 항목 | 설계 기준 |
|---|---|
| 텍스트 | `Flowify` |
| 정렬 | 중앙 |
| 폰트 크기 | 데스크톱 `48px` |
| 폰트 두께 | `medium` |
| 색상 | `text.primary` |

모바일에서는 화면 폭에 따라 `40px` 또는 `36px`까지 축소 가능하다.

### 5-3. 로그인 카드

카드는 최소 요소만 담는 흰색 표면 카드로 구성한다.

| 항목 | 설계 기준 |
|---|---|
| 최대 너비 | `437px` 기준 |
| 실제 구현 | `w="full"` + `maxW="437px"` |
| 배경 | `bg.surface` |
| border | `1px solid #f2f2f2` |
| radius | `20px` |
| shadow | `0 4px 4px rgba(0, 0, 0, 0.25)` |
| 내부 패딩 | 데스크톱 `48px`, 모바일 `32px` |

### 5-4. 카드 내부 구성

카드 내부 순서는 아래와 같다.

1. 제목 `로그인`
2. Google 로그인 버튼
3. 에러 메시지 (오류 발생 시에만)

설명 문구는 1차 구현에서 제거한다.

### 5-5. Google 로그인 버튼

버튼은 흰 배경 + border 조합의 단일 버튼으로 구현한다.

| 항목 | 설계 기준 |
|---|---|
| 배경 | `bg.surface` |
| border | `1px solid #efefef` |
| 너비 | 카드 전체 너비 |
| 내부 정렬 | 아이콘 + 텍스트 수평 중앙 정렬 |
| gap | `10px` |
| 텍스트 | `Google 로그인` |
| 아이콘 | Google 컬러 아이콘 1개 |

### 5-6. 반응형 기준

모바일에서는 아래 정도만 축소한다.

| 항목 | 데스크톱 | 모바일 |
|---|---|---|
| 워드마크 크기 | `48px` | `36~40px` |
| 카드 패딩 | `48px` | `32px` |
| 카드 폭 | `437px` | `100%` |
| 외곽 여백 | 넉넉함 | `24px` 이상 유지 |

레이아웃 구조 자체는 모바일에서도 그대로 유지한다.

## 6. 동작 설계

### 6-1. 로그인 시작

버튼 클릭 시 기존 로직을 유지한다.

```text
1. 에러 메시지 초기화
2. pending 상태 true
3. buildGoogleLoginStartUrl() 호출
4. window.location.href 이동
```

### 6-2. 실패 처리

`buildGoogleLoginStartUrl()` 단계에서 예외가 나면 아래 메시지를 유지한다.

```text
구글 로그인 설정 확인이 필요합니다.
```

에러 메시지는 카드 하단에만 조건부로 노출한다.

### 6-3. pending 상태

pending 상태에서는 아래 규칙을 따른다.

- 버튼 비활성화
- 라벨을 `구글 로그인 이동 중...` 으로 변경
- 중복 클릭 방지

## 7. 구현 파일 제안

이번 화면 구현에 직접 관련된 파일은 아래와 같다.

| 파일 | 역할 |
|---|---|
| `src/pages/login/LoginPage.tsx` | 로그인 페이지 메인 구현 |
| `src/pages/login/index.ts` | 페이지 export |
| `src/shared/libs/auth-redirect.ts` | `buildGoogleLoginStartUrl()` 유지 |
| `docs/LOGIN_PAGE_DESIGN.md` | 현재 설계 문서 |

### 7-1. 구현 방향

현재 페이지는 단일 파일로도 충분히 구현 가능하다.

다만 아래 조건이 생기면 컴포넌트 분리를 고려한다.

- 버튼 스타일을 다른 로그인 화면에서도 재사용할 때
- 브랜딩 헤더를 공통 인증 페이지 UI로 뽑고 싶을 때

1차 구현은 `LoginPage.tsx` 단일 파일 유지가 가장 단순하다.

## 8. 검증 기준

### 8-1. 시각 검증

| 항목 | 기대 결과 |
|---|---|
| 워드마크 위치 | 카드 상단 중앙에 독립 배치 |
| 워드마크와 카드 간격 | 시안과 유사한 넓은 간격 유지 |
| 카드 폭 | 데스크톱에서 지나치게 넓지 않음 |
| 카드 내부 밀도 | 설명 문구 없이 단순함 유지 |
| 버튼 스타일 | 검정 CTA가 아닌 border형 버튼 |
| 전체 인상 | 화면 중앙에 브랜드명과 로그인 카드만 보이는 구조 |

### 8-2. 기능 검증

| 항목 | 기대 결과 |
|---|---|
| 버튼 클릭 | Google 로그인 시작 URL로 이동 |
| pending 상태 | 버튼 비활성화 및 문구 변경 |
| 예외 처리 | 에러 메시지 노출 |
| 반응형 | 모바일 폭에서 카드가 화면 밖으로 나가지 않음 |

## 9. 후속 작업

### 9-1. 바로 이어질 작업

1. `/auth/callback` 페이지와의 시각 톤 일관성 검토
2. 실제 브라우저 해상도별 수동 QA
3. Google 아이콘을 Figma 자산으로 교체할지 검토
4. 남은 설정/문서 변경사항 정리

### 9-2. 추후 보강 작업

- 로그인 페이지와 `/auth/callback` 페이지의 톤 일관성 정리
- 공통 인증 화면 레이아웃 필요 여부 검토
- 접근성 기준으로 버튼 focus 스타일 보강
- 브랜드 워드마크를 텍스트 대신 자산으로 교체할지 검토
