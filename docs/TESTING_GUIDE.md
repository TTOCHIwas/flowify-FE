## 4. 시작/도착 노드 설정 위자드

> **v4.1 설계:** 시작/도착 노드의 전체 위자드(카테고리 → 서비스 → 요구사항 → 인증)는 ServiceSelectionPanel에서 완료된다. SSP는 내부에 별도 placeholder/서비스 preview를 만들지 않고, **실제 캔버스 placeholder 또는 미완료 노드 기준으로 카드가 오른쪽 48px에 anchor**된다. Step 전환 시 패널 외곽은 고정되고 카드 내부 내용만 바뀐다.

## 5. 중간 노드 설정

> **v4.3 설계:** 중간 노드는 placeholder 클릭 시 **듀얼 패널(InputPanel + OutputPanel)**이 즉시 열린다. 위자드 단계가 **OutputPanel 오른쪽 패널 안에서** 진행된다. 별도 ChoicePanel 오버레이는 없다.

### 5.1 중간 노드 추가 — 듀얼 패널 위자드 진입

| # | 테스트 | 기대 결과 | 결과 |
|---|--------|-----------|------|
| 5.1.1 | "다음" placeholder 클릭 | **듀얼 패널** 열림 (왼쪽: InputPanel, 오른쪽: OutputPanel 위자드 모드) | |
| 5.1.2 | 왼쪽 패널 (InputPanel) | 이전 노드의 들어오는 데이터가 표시됨 | |
| 5.1.3 | 오른쪽 패널 (OutputPanel) | "설정" 헤더 + 위자드 첫 번째 단계 UI | |
| 5.1.4 | SSP(ServiceSelectionPanel)가 열리지 않는지 | SSP는 시작/도착 placeholder에서만 열림 | |
| 5.1.5 | 패널 사이 노드 체인 | 이전 노드 → 현재 노드 → 다음 노드만 보임, 다른 노드는 숨김 | |

### 5.2 Step 1 — 처리 방식 (OutputPanel 내부, 목록형 데이터일 때만)

| # | 테스트 | 기대 결과 | 결과 |
|---|--------|-----------|------|
| 5.2.1 | 이전 노드 출력이 file-list | 오른쪽 패널에 "파일들을 어떻게 처리할까요?" + 선택지 표시 | |
| 5.2.2 | 이전 노드 출력이 single-file | 처리 방식 단계 생략, 바로 액션 목록 표시 | |
| 5.2.3 | "한 파일씩" 선택 | LOOP 노드 생성 → SINGLE_FILE 액션 목록 표시 (Step 2) | |
| 5.2.4 | "전체를 하나로 합쳐서" 선택 | 노드 미생성, 현재 타입 유지 → actions 비어있으면 완료 | |

## 테스트 진행 방법

1. **순서대로 진행**: 2장(캔버스) → 3장(Placeholder) → 4장(위자드) → 5장(중간 노드) → 6장(삭제) → 7장(닫기 규칙) → 8장(레이아웃) → 9장(Edge) → 10장(툴바) → 11장(라우팅)
2. **결과 기록**: 각 항목의 "결과" 칸에 `✅ Pass` / `❌ Fail (설명)` 기록
3. **Fail 발견 시**: 즉시 스크린샷 + 콘솔 에러 캡처
4. **매 섹션 완료 후**: 페이지 새로고침하여 상태 초기화 후 다음 섹션 진행
